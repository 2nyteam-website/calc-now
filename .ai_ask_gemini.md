# CalcNow UI/UX 및 전체 검증 요청

## 사이트 정보
- URL: 방금 배포 완료 (Vercel)
- 도메인: calcnow.cc (미구매, 추후 연결)
- 기술: Next.js + Tailwind + shadcn/ui + TypeScript
- 100% 클라이언트 사이드, Vercel 무료 호스팅
- 10개 계산기: compound-interest, loan, mortgage, percentage, bmi, salary, tip, age, discount, unit-converter

## 사용자 피드백
- "사이트가 불안정해 보인다"
- "계산기랑 어울리지 않는다"
- "어두워서 잘 안 보인다" (다크모드 기본)
- "시각적인 걸 많이 놓친 것 같다"

## 기존 사이트 참고
- DevToolKit (devtoolkit.cc): 개발자 도구 → 다크모드가 어울림
- ConvertFree (convertfree.cc): 파일 변환 → 다크모드가 어울림
- CalcNow: 계산기 → 일반인 타겟, 다크모드가 어울리지 않을 수 있음

## 검증 요청 사항

### 1. UI/UX 전반 (가장 중요!)
- 다크모드 vs 라이트모드: 계산기 사이트에 어떤 것이 더 적합한가?
- 색상 팔레트: 금융 계산기에 어울리는 색상 조합 추천
- 숫자/결과 표시: 금액, 퍼센트 등 계산 결과의 가독성이 충분한가?
- 입력 필드: 크기, 간격, 레이블 가독성
- 결과 영역: 시각적으로 충분히 돋보이는가?
- 차트/그래프: 대출 상환 스케줄, 복리 성장 등 시각화가 필요한가?
- 전반적인 "계산기 느낌"이 나는가? (참고: calculator.net, omnicalculator.com)

### 2. 경쟁사 대비 분석
- calculator.net, omnicalculator.com, calcul.io 등과 비교했을 때 UI 수준은?
- 이 사이트들이 잘하는 점 중 우리가 놓친 것은?

### 3. 금융 계산기 특화 UX
- 대출/모기지 계산기에 상환 스케줄 테이블이 필요한가?
- 복리 계산기에 성장 차트가 필요한가?
- 숫자 포맷팅 (천 단위 콤마, 소수점 2자리 등)이 제대로 되어 있는가?
- 슬라이더(Range Input)가 필요한 계산기가 있는가?

### 4. SEO/콘텐츠
- 각 계산기 하단 SEO 텍스트가 충분한가? (AdSense 승인 기준 300~500자)
- 놓친 메타 태그나 구조화 데이터가 있는가?

### 5. 보안/성능
- 빠진 보안 헤더가 있는가?
- 성능 최적화할 부분이 있는가?

### 6. 추가 계산기 추천
- RPM 높은 키워드 기반으로 추가하면 좋을 계산기 5~10개 추천

각 항목에 구체적으로 답변하고, 즉시 수정해야 할 사항과 나중에 해도 되는 사항을 구분해주세요.
